# 🦜 鸟类音频识别系统 - 功能规格书 (spec.md)

> **源头声明 (Source of Truth)**：本规格书定义系统“应该做什么”（What），为整个设计与代码实现的绝对真理来源。**严禁在本文档中写入任何特定技术框架、数据库选型或代码逻辑细节**。

---

## 📖 1. 业务愿景 (Vision)
构建一个**直观、沉浸且极速响应**的鸟类声学分析与科普平台。
地球上的鸟类鸣叫千变万化，用户可以自由地通过**实时录音**或**音频上传**，让系统协助充当“自然调音师”，剥离杂音，快速检索并图谱化展示出周围环境中正在啼叫的鸟类物种及对应的置信百分比。

---

## 👥 2. 目标用户 (Target Audience)
*   **观鸟爱好者 (Bird Watcher)**：在户外听到陌生鸟叫，希望通过手机录音快速识别，提升观鸟日志的精确度。
*   **自然科学教育者 / 学生**：用于室外考察或生物课堂展示，需要清晰、卡片化的视觉呈现。
*   **声学研究/环保人员**：收集生态保护区内的音频监控，用来间接评估生物多样性丰富度。

---

## 🗺️ 3. 用户旅程与心流 (User Journey)

### 🟢 Phase A：驻足与启航 ( Landing )
1.  用户进入主界面，映入眼帘的是**沉浸式自然动效**（如森林风、晨曦微光等），消除电子产品的冰冷感。
2.  提供两个核心操作：
    *   **【点击录制】**：激活设备的麦克风传感器，开启捕获环境音的心流。
    *   **【文件拖拽/上传】**：支持将本地录制的音频一键直投。
3.  **可选参数（地理位置锚点）**：系统询问是否允许获取当前 GPS 坐标。其业务意图是利用地理气候特征，缩小识别盲区，大幅提升准确率。用户可拒绝，不影响基本链路。

### 🟡 Phase B：聆听与沉思 ( Recording & Analyzing )
1.  **录音中**：
    *   界面展示**波形脉冲/声纹动画**，给予用户“系统正在认真捕捉每一丝频率”的极佳视觉反馈。
    *   支持**暂停、取消、重置**，底部提供录制时长计时。
2.  **分析中**：
    *   用户点击“分析”后，进入过渡渲染屏。
    *   播放轻柔的微动画（例如：一只鸟在树枝上跳动或波纹扩散）。
    *   展示**骨架屏或智能提示语**（如：“正在剥离背景风声...”、“正在匹配鸟类声频指纹...”），消除等待焦虑度。

### 🔴 Phase C：洞察与揭晓 ( Result Dashboard )
1.  **分析总结头部**：大红底色或高亮色框，第一眼呈报“共捕获 X 种鸟叫”。
2.  **核心物种卡片 (Specie Card)**：
    *   **命名**：展示该鸟类的俗名（Common Name）与学术名（Scientific Name）。
    *   **置信概率 (Confidence)**：以百分比数据 ➕ 进度条/环状彩条显示系统确信的程度。
3.  **时间轴时间线 (Timeline)**：
    *   折叠或展开的时间轴，清晰标柱：`00:12 - 00:15` 在环境中听见了该鸟。
4.  **物种百科互通 (可选升级)**：点击卡片可跳转或翻转展示该鸟类的羽色、习性或生态照片。

## 🛠️ 4. 业务验收标准 (Acceptance Criteria)

### ✅ AC 1：零门槛摄取 (Zero Friction Intake)
*   系统必须支持免登录直接录音，且音频文件可以在不超过 3 次点击内直接从桌面拖入上传。
*   支持录音长达至少 2 分钟，能充分捕获鸟类多变的循环啼叫。

### ✅ AC 2：经纬度辅助算力降级策略
*   **策略一**：若地理位置可用，优先按照当前区域生态特征给出精确结果。
*   **策略二（兜底）**：若按地理区域匹配**无任何有效物种**，系统需**自动且静默地**扩大算力范围至“全球通识标准”，再一次检索并返回结果，防止用户面对空白屏。

### ✅ AC 3：视觉心流一致性
*   从结束音频到吐出结果面板，分析状态必须有明确的阻断式过度，且分析耗时不可在没有任何提示语的情况下超过 30 秒（若超时需降级容错）。

## 🚨 5. 异常与边缘安全案例 (Exceptions & Edge Cases)

| 场景边界 | 理想响应 (Expected Behavior) | 用户负反馈解决方案 |
| :--- | :--- | :--- |
| **全静音音频 / 只有白噪** | 分析结果为空。 | 界面不报 Critical 报错，而是温馨提示：“似乎周围很安静？可以尝试录制离树梢更近的声音。” |
| **超长超大音频文件** | 文件过重。 | 在进入分析前，立刻在界面阻止，并提示：“系统暂时只支持 210MB 级内的文件，您可以截取精彩片段。” |
| **麦克风权限被绝对拒绝** | 无法录制。 | “点击录音”置灰，悬浮提示：“系统需要您的麦克风授权来充当自然调音师，请在浏览器设置中开启。” |
| **分析中突然断网** | 发送请求未遂。 | 系统在 5 秒内抓捕到断联，停止 Loading 动效，展示带有 “重试 (Retry)” 按钮的离线卡片。 |
| **同时出现几十种低置信鸟鸣** | 噪音环境干扰。 | 首屏只展示置信度 > 30% 的前三名大类，其余折叠在“其他低音量检测”中，保证视觉清爽。 |
